# 联调前置信息清单与缺口台账

> 更新时间：2026-04-01。本文档服务后续执行型子 Agent，用于在开始真实接口联调前快速判断：仓库内已经确认了什么、还缺什么、下一步应向哪里补信息。它只在“未来需要真实接口替换”这一场景下使用，不代表当前仓库已经进入真实联调阶段。

## 1. 使用方式

- 先看 `A. 仓库内已确认事实`，不要重复在仓库里寻找已经明确不存在的内容。
- 再看 `B. 仓库内无法确认的信息`，这些信息需要向后端仓库、接口文档、环境维护人或业务同学补齐。
- 如果任务仍停留在页面、mock、service、localStorage 范围内，不要把本页当成“当前已联调”的证明材料。
- 开始执行具体联调任务前，优先把本页涉及目标模块的缺口补齐，而不是直接进入页面改造。

## 3. B. 当前仓库无法确认、必须向外部补充的信息

### 3.1 后端与环境

- 后端仓库位置、分支、负责人。
- 联调环境地址、网关地址、各服务 base URL。
- 本地/测试/预发环境的环境变量模板。
- 是否存在 API 网关、BFF、代理规则或跨域配置。

### 3.2 接口与鉴权

- 真实接口文档、Swagger/OpenAPI 地址或接口清单。
- 登录方式、token/cookie 鉴权方案、权限头要求。
- 菜单、按钮、角色权限与后端接口/资源的真实映射关系。
- `settings/sync` 中出现的 `/api/v1/*/sync` 是否真实存在、是否仍沿用该命名。

### 3.3 数据与验证

- 数据库类型、库表结构、核心表关系、字段口径。
- 各业务页面与后端字段映射关系。
- 联调测试账号、角色、组织、仓库、店铺、供应商等基础数据。
- 每个目标页面的真实闭环验收标准和优先级。

## 4. 后续收集清单

执行型子 Agent 只有在明确进入真实接口替换任务前，才需要补齐以下项目中的目标页相关部分：

1. 后端仓库或接口文档入口。
2. 环境地址与鉴权方式。
3. 目标页面对应的接口清单与字段映射。
4. 测试账号与基础数据准备方式。
5. 该页面当前是“已有 service 可替换数据源”还是“仍需先从页面内嵌 mock 中抽离”。

## 5. 适合直接交给执行型子 Agent 的结论

- 可在本仓继续推进：`settings`、`organization`、`logistics`、`supply-chain/forecast` 等已有 service 的页面梳理、替换点标注、页面与 mock/service 链路核查。
- 可在本仓继续推进：`finance`、`hr`、`project`、`procurement`、`announcement` 等模块的页面原型盘点、内嵌 mock 抽离评估、路由与菜单链路梳理。
- 必须依赖外部后端信息后才有意义：真实接口接入、真实数据库字段映射、鉴权接入、联调执行与“已联调完成”判定。
- 如果外部连后端仓库、环境地址、接口文档都还没有，本轮不适合直接进入真实联调执行。

## 6. 文档维护约定

- 能在仓库内直接核实到的内容，更新到第 2 节。
- 无法在仓库内核实、必须依赖外部信息的内容，放在第 3 节。
- 当某个未知项被补齐后，优先把它从“缺口”移动到“已确认事实”，不要重复保留在两边。
