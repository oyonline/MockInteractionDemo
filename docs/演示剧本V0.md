## 1. 演示目标

本次演示要让观看者在 10 分钟内理解：

1. **这是一个全局视角高保真交互原型**，不是后端真实系统、不是 ERP 上线版
2. **所有 14 个一级业务模块都能看、都能进、都能点**（首页 14 卡零 PlaceholderPage）
3. **关键操作有反馈**（Toast / ConfirmDialog 全仓统一，无原生 alert / window.confirm）
4. **3 条关键流程已具备原型级闭环**：
   - **销量预测推送**（SalesForecastPage：草稿→已推送计划，刷新仍在）
   - **审批通过/驳回**（ApprovalCenter/Detail：状态+时间线+待办联动，刷新仍在）
   - **基于预测生成供应计划**（SupplyPlanPage：跨页面消费 SalesForecast 的快照，刷新仍在）
5. **8 个 overview 已升级为轻量看板**（KPI + 状态分布 + 最近动态 + 子页导航 + 底部判断条）
6. **跨模块数据已开始联动**：销售预测推送数 → 供应链 overview KPI；审批操作 → 财务 overview KPI；供应计划生成 → 采购 overview 动态
7. **适合用于产品蓝图展示与后续真实开发参考**

**不要试图证明的事**：
- ❌ 这不是真实 SaaS 系统（无登录、无权限、无后端、无数据库）
- ❌ 不是用真实交易数据驱动（mock + localStorage）
- ❌ 不演示角色切换（用户已明确放弃）

```bash

```

如要体验"刷新仍在"效果，可先 `npm run build && serve -s build` 用静态服务器演示。

### 2.3 演示前数据初始化（重要）

**强烈建议演示前 5 分钟做一次"数据预热"**，让 mockStore 中各 key 都有真实快照：

| 顺序 | 动作 | 目的 |
|---:|---|---|
| 1 | 打开浏览器 DevTools → Application → Local Storage → 清空所有 `ecommerce:*` key | 干净起点（**可选**，不清也行） |
| 2 | 点首页"欧美事业部" → 左侧菜单"销量预测" | 触发 SalesForecastPage 首次挂载，把 `ecommerce:sales:forecast-page-state` 固化 |
| 3 | 点首页"工作台→待办&审批"或顶部 Bell | 触发 ApprovalCenterPage，把 `ecommerce:approvals` 固化 |
| 4 | 点首页"供应链计划→供应计划" → 点"基于销售预测生成"按钮一次 | 触发 SupplyPlanPage，生成一批 `ecommerce:supply-chain:supply-plans` 真实数据 |
| 5 | 点击物流商列表（`/logistics/vendors`）一次 | 触发 logisticsService 把 `ecommerce:logistics:*` 固化（可选；LogisticsOverview 用） |
| 6 | 回首页 | 准备就绪 |

> **跳过预热的影响**：演示中 Step 4（推送预测）会成为"第一次写入"，故事仍然成立；但 Step 5（供应计划 overview）的 KPI 数字会是 0，需要先去 SupplyPlanPage 点击生成一次。

### Step 4 · 销售预测推送 V3（核心动作 1） (1:30)

- **页面路径**：`/sales/us/forecast`（从 overview 子页"销量预测"卡进，或左侧菜单点）
- **演示动作**：
  1. 进入 SalesForecastPage（1342 行重型实现）
  2. 顶部展示版本管理区：当前版本徽章（如"草稿1"或"已推送计划"）+ 版本时间线
  3. **如果当前版本是草稿**：点页面下方"推送至计划"按钮
  4. 弹出 ConfirmDialog（演示原型已统一替换 window.confirm）
  5. 确认 → 版本徽章立即变为"已推送计划" + 状态变 `draft_editing → pending_plan` + 右侧时间线增加一条 + 顶部 Toast"已推送至计划确认"

- **预期系统反馈**：
  - 版本头部徽章颜色从 warning 灰变为 brand 蓝
  - 时间线新增一条 publishedAt 记录（当前时间）
  - Toast 右上角弹出 success 绿条，3 秒自动消失
  - mockStore[SALES_FORECAST_PAGE_STATE] 写入；F5 后状态仍在
- **推荐讲解话术**：
  > "销售经理基于市场判断推送预测。注意我点击「推送至计划」时弹出的是统一的确认弹窗，不是浏览器原生 confirm。确认后版本状态立即变化，时间线增加一条记录，下游供应链可以在他们的页面看到这个信号。我现在按 F5 刷新一下 — 看，状态保留了，这是真实的 mock 闭环。"
- **注意事项**：
  - SalesForecastPage 是 1342 行的重型页，加载略慢，**首次进入留 1-2 秒** 不要急着点
  - 如果当前版本已经是"已推送计划"（演示前已操作过），可以演示**版本切换**（点其他月份的草稿）+ 或演示**单元格内联编辑保存**（点表格单元格输入新值，Toast"保存成功"）
- **失败兜底**：如果 ConfirmDialog 没出来，可以直接讲"原型阶段所有不可逆操作都用统一的 ConfirmDialog 拦截"，并在 SettingsBasicPage 演示一次（点恢复默认按钮）。
