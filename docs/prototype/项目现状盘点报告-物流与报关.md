# 项目现状盘点报告（物流与报关模块准备）

> 扫描范围：路由与多标签页、页面 UI 通用模式、组件复用、样式与布局约束。结论均附带**文件路径 + 关键标识符**。

---

## 1) 路由与多标签页机制

### navigationConfig 位置与 tab 数据结构

- **navigationConfig** 定义在 `src/App.js` 第 59～191 行，常量数组，顶层/子级结构为：
  - 顶层项：`id`、`name`、`icon`、`path`（叶子）或 `children`（子菜单）
  - 子项：`id`、`name`、`path`
- **Tab 数据结构**（单标签）：`{ id, name, path, data? }`
  - `id`：唯一标识，可为 path 或自定义（如 `product/sku-detail-xxx`）
  - `name`：标签标题
  - `path`：用于 `renderTabContent` 的 switch 分支
  - `data`：可选，传给详情类页面的记录（如 `tab.data`）

**文件路径**：`src/App.js` — `navigationConfig`、`tabs` 初始值 `useState([{ id: '/home', name: '首页', path: '/home' }])`

### 打开 / 关闭 / 切换标签页

- **打开新标签页**：`openTab(tabInfo)`，`tabInfo` 含 `id, name, path, data`；若已存在同 `id` 则只激活不新增。
  - **位置**：`src/App.js` 第 358～368 行，`openTab`（useCallback）
- **从侧栏打开**：`handleNavigate(path, name)`，同 path 则激活，否则 `setTabs(prev => [...prev, { id: path, name, path }])`。
  - **位置**：`src/App.js` 第 371～379 行，`handleNavigate`
- **关闭标签页**：`closeTab(tabId)`，过滤掉该 tab，若全关则保留首页。
  - **位置**：`src/App.js` 第 386～392 行，`closeTab`
- **切换标签页**：`switchTab(tabId)` → `setActiveTabId(tabId)`，传给 TabBar 的 `onTabClick`。
  - **位置**：`src/App.js` 第 400～402 行，`switchTab`；第 596～597 行 `onTabClick={switchTab}`

### 跨页联动（列表→详情在新 tab 打开）

- **约定**：列表页通过 **`onOpenDetail`** 回调打开详情；由 App 在 `renderTabContent` 里注入，传入的是 **`openTab`**（或包装后的函数）。
- **调用方式**：列表页调用 `onOpenDetail({ id, name, path, data })`，与 `openTab` 入参一致。
- **已有用法**：
  - `SupplierListPage`：`onOpenDetail` 传入，点击「详情」时调用 `onOpenDetail({ id: 'supplier-detail-'+supplier.id, name: '...', path: '/procurement/supplier/detail', data: supplier })`  
    **位置**：`src/pages/SupplierListPage.js` 第 5 行入参、第 286～295 行 `handleViewDetail`
  - App 中：`case '/procurement/supplier': return <SupplierListPage onOpenDetail={openTab} />`  
    **位置**：`src/App.js` 第 508 行
  - 其他：`ExpenseFactPage` / `ExpenseApprovalListPageSimple` 用 `onOpenDetail(rec)`，`BudgetVersionPage` 用 `onOpenDetail={openTab}`；Settings 部分用 `onOpenTab`。

**结论**：跨页联动统一用 **`onOpenDetail`**（或 `onOpenTab`），入参为 `{ id, name, path, data }`，由 App 注入 `openTab` 即可。物流与报关新列表页若需「在新 tab 打开详情」，应接收 `onOpenDetail` 并同样传入上述对象。

## 2) 页面 UI 的通用模式

### 标准列表页模板

- **推荐参考**：`src/pages/SupplierListPage.js`
  - 结构：顶部统计卡片 → 筛选区（标题 + 主按钮 + 筛选项 + 重置）→ 表格区（flex-1 + overflow-auto + table）→ 底部分页条。
  - 表格容器：`flex-1 bg-white rounded-lg shadow-sm flex flex-col overflow-hidden`，内部 `flex-1 overflow-auto` + `table`。
  - **注意**：SupplierListPage 底部分页是**内联实现**（第 514～539 行），未使用 `TablePagination` 组件；与使用 `TablePagination` 的页面（见下）风格一致，可二选一。

### 表格：最小宽度、sticky header、overflow

- **最小宽度**：表格用 `min-w-[Npx]`，无全局常量，常见：`min-w-[1100px]`～`min-w-[1600px]`（如 SupplierListPage `min-w-[1300px]`，UserManagementPage `min-w-[1600px]`）。
- **Sticky 表头**：`<thead className="bg-gray-50 sticky top-0">` 或 `sticky top-0 z-10`（在可滚动容器内）。
- **Overflow 容器**：多为「外层 flex-1 + 内层 overflow-auto」：
  - `div.flex-1.overflow-auto` 包住 `table`；
  - 或 `div.flex-1.min-h-0.overflow-auto`（在 flex 列布局中避免撑爆）。

**典型组合**（多处复用）：
- `src/pages/SupplierListPage.js` 第 455～458 行：`flex-1 overflow-auto` + `table min-w-[1300px]` + `thead sticky top-0`
- `src/pages/UserManagementPage.js` 第 448～450 行：`flex-1 overflow-auto min-h-0` + `table min-w-[1600px]` + `thead sticky top-0 z-[1]`
- `src/pages/settings/SettingsEnumRulePage.js` 第 223～226 行：`flex-1 min-h-0 overflow-auto flex flex-col` + 内层 `overflow-auto flex-1 min-h-0` + `thead sticky top-0`

### 分页：state / 计算 / 渲染

- **State**：当前页通常为 `currentPage`（从 1 开始），部分页面用 query 对象（如 `appliedQuery.page`）。
- **计算**：`totalPages = Math.max(1, Math.ceil(total / pageSize))`，`paginatedData = useMemo(() => data.slice((currentPage-1)*pageSize, currentPage*pageSize), [...])` 或等价逻辑。
- **渲染**：要么用 **`TablePagination`**（见下），要么与 SupplierListPage 一样在表格卡片底部写一层 `p-4 border-t bg-gray-50 flex justify-between` + 上一页/下一页 + 共 N 条。

**使用 TablePagination 的页面**：`SettingsDictPage`、`SettingsApiSyncPage`、`SettingsParamsPage`、`SettingsLogPage`、`SettingsSchedulerPage`、`SettingsEnumRulePage`、`UserManagementPage`。

### Modal / Drawer 实现模式

- **Modal**：
  - 遮罩 + 内容：`<div className="fixed inset-0 z-50 flex items-center justify-center">`，内层 `<div className="absolute inset-0 bg-black/30" onClick={onClose} />`，再内层 `relative bg-white rounded-lg shadow-xl ... max-h-[90vh] overflow-hidden`。
  - 内容区滚动：`overflow-auto flex-1` 放在 body 区域；底部按钮区单独 `border-t` + `flex justify-end gap-2 p-6`。
  - **位置**：`src/pages/SupplierListPage.js` 第 588～601 行；`src/pages/UserManagementPage.js` 第 537～542 行；`src/pages/settings/SettingsEnumRulePage.js` 第 277～283 行。
- **Drawer**：`StoreDeptEditStoreDrawer.js` 使用 `createPortal(..., document.body)` 挂到 body，避免被父级 overflow 裁剪；`fixed inset-0` 遮罩 + 侧边滑出层。
  - **位置**：`src/pages/StoreDeptEditStoreDrawer.js` 第 93 行、第 347 行

## 4) 样式与布局约束

### main 内容区容器

- **主内容区**：`<main className="flex-1 min-h-0 overflow-auto p-6">`，在 App 的「主内容区」flex 列内。
  - **位置**：`src/App.js` 第 601～602 行

### 页面内部撑满与滚动

- **常见**：页面根节点 `flex flex-col h-full min-h-0`，中间内容区 `flex-1 min-h-0 overflow-auto`，保证在 flex 布局下不撑爆且可滚动。
- **列表页**：表格外层 `flex-1 ... overflow-hidden`，表格所在 div `flex-1 overflow-auto`，使表头 sticky 生效。
- **位置**：如 `UserManagementPage` 根 `flex flex-col h-full`，表格区 `flex-1 overflow-auto min-h-0`；`RolePermissionPage` 多处 `flex-1 min-h-0 overflow-auto`。

### overflow 裁剪与下拉菜单定位

- **已知问题**：表格在 `overflow-auto` 的容器内时，行内 **绝对定位的下拉/详情菜单** 会被裁剪或错位（clipping 父级为带 overflow 的 div）。
- **解决方式**：操作列使用 **`ActionBar`**，其「更多」菜单通过 **`ReactDOM.createPortal(..., document.body)`** 挂到 body，用 **position: fixed** + **getBoundingClientRect** 定位，避免在 overflow 容器内渲染。
  - **位置**：`src/components/ActionBar.js` 第 3 行注释、第 114～117 行 `createPortal`、第 59 行 `style={{ top, left }}` fixed
- **若自定义下拉**：要么也用 Portal 挂 body + fixed，要么参考 ActionBar 的实现；不要在图内 `absolute` 的父级带 `overflow-auto` 的 div 里渲染下拉。

**常见易踩坑的 class 组合**：
- `flex-1 overflow-auto min-h-0`：正确，用于可滚动区域；
- `flex-1 min-h-0` 且子元素无 `overflow-auto`：子内容可能把父撑开；
- 在 `overflow-auto` 的 div 内使用 `position: absolute` 的下拉：会被裁切，需改为 Portal + fixed。

## 6) 对物流与报关模块新增页面的建议开发方式

| 建议 | 说明 |
|------|------|
| **复制模板** | 以 **SupplierListPage** 为模板复制一份，改路由、数据源、列与筛选项；底部分页可改用 **TablePagination** 统一风格（传入 `itemName` 如「条」/「单」）。 |
| **路由与联动** | 在 `renderTabContent` 中为每个需实现的 path 增加 `case '/logistics/xxx': return <XxxPage />`；若列表需打开详情 tab，App 传入 `onOpenDetail={openTab}`，列表内调用 `onOpenDetail({ id, name, path, data })`。 |
| **表格与布局** | 表格外层 `flex-1 overflow-auto`（或 `flex-1 min-h-0 overflow-auto`），table `min-w-[1100px]` 等，thead `bg-gray-50 sticky top-0`；操作列若有多项用 **ActionBar**，避免行内下拉被裁切。 |
| **Modal** | 沿用现有模式：`fixed inset-0 z-50 flex items-center justify-center` + 遮罩 `bg-black/30` + 内容区 `overflow-hidden`/`overflow-auto` + 底部按钮区。 |
| **不额外抽组件** | 表格仍用原生 `<table>`，分页用现有 **TablePagination**；仅当多个物流页出现重复复杂块（如「物流单头+明细」）时再考虑抽成页面级子组件，无需先抽象通用表格组件。 |

**总结**：优先 **复制 SupplierListPage** 做列表页，分页用 **TablePagination**，操作列用 **ActionBar**，详情通过 **onOpenDetail** 打开新 tab；仅在做多页重复块时再考虑抽组件，不做大范围重构。
