## 3) 页面实现模式

### 列表页统一结构

所有列表页遵循统一模式（以 `LogisticsVendorListPage.js` 为例）：

```
页面结构
├── 统计卡片区 (grid grid-cols-4)
├── 筛选区（关键词、状态等）
├── 操作按钮区（新增）
├── 表格区 (flex-1 overflow-auto)
│   └── table (min-w-[1200px])
│       └── thead (sticky top-0)
└── 分页区 (TablePagination)
```

### 详情页统一结构

详情页采用卡片式布局：

```
详情页结构
├── 顶部操作栏（返回、编辑、删除）
├── 基础信息卡片（只读展示）
├── 扩展信息卡片（标签页/分区块）
└── 关联数据表格（如渠道列表）
```

### 跨页联动

列表页 → 详情页通过 `onOpenDetail` 回调实现：

```javascript
// 列表页调用
onOpenDetail({
  id: `logistics-vendor-${vendor.id}`,
  name: `物流商详情: ${vendor.name}`,
  path: `/logistics/vendors/${vendor.id}`,
  data: vendor
});
```

## 4) 技术实现亮点

### ActionBar 组件

行内操作下拉菜单使用 `ActionBar` 组件，通过 `createPortal` 挂到 body，避免被表格 `overflow` 裁剪。

### 动态面包屑

详情页面包屑根据数据动态生成，支持返回列表。

### 表单验证

编辑表单使用原生 HTML5 验证 + 自定义验证逻辑，错误提示统一样式。
