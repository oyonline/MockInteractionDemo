## 2. 技术架构


## 4. 路由与导航

### 4.1 路由方案

- **无 React Router**：使用基于 `path` 的 switch/case 实现
- **多标签页系统**：所有页面以标签形式打开，支持多开
- **动态侧边栏**：根据当前模块显示不同导航（`DynamicSidebar.js`）

### 4.2 路由配置位置

1. **导航定义**：`src/layouts/DynamicSidebar.js` 的 `navConfig`
2. **路由渲染**：`src/App.js` 的 `renderTabContent()` 函数

### 4.3 首页入口顺序

```javascript
// 当前首页入口顺序（mainEntries）
1.  经营管理分析
2.  欧美事业部
3.  亚太事业部
4.  产品中心
5.  项目管理
6.  KK-US私域管理
7.  供应链计划
8.  供应链采购
9.  供应链物流
10. 供应链质量
11. 财务中心
12. 人力资源
13. 权限设置
14. 系统设置
```

### 4.4 添加新页面的步骤

1. 创建页面组件：`src/pages/[模块]/NewPage.js`
2. 在 `App.js` 中导入组件
3. 在 `DynamicSidebar.js` 的 `navConfig` 中添加导航项
4. 在 `App.js` 的 `renderTabContent()` 中添加 case 处理

## 5. 开发约定

### 5.1 代码规范

- **组件命名**：PascalCase（如 `UserManagementPage`）
- **文件命名**：页面组件用 PascalCase，工具函数用 camelCase
- **样式**：使用 Tailwind CSS 工具类，无 CSS Modules
- **图标**：统一使用 `lucide-react`

### 5.2 列表页标准结构

参考 `SupplierListPage.js`：

```
页面结构（从上到下）
├── 统计卡片区 (grid grid-cols-4)
├── 筛选查询区 (flex items-center gap-4)
├── 操作按钮区 (新增/导入/导出)
├── 表格区 (flex-1 overflow-auto)
│   └── table (min-w-[1200px])
│       └── thead (sticky top-0)
└── 分页区 (TablePagination)
```

### 5.3 表格与分页

- **表格**：原生 `<table>` + Tailwind，最小宽度 `min-w-[1100px]`～`min-w-[1600px]`
- **Sticky 表头**：`<thead className="bg-gray-50 sticky top-0">`
- **分页组件**：使用 `TablePagination`（已有组件）
- **操作列下拉**：使用 `ActionBar`（Portal 挂 body，避免裁剪）

### 5.4 Mock 与 Service

```
页面 → Service → Mock 数据
         ↓
    localStorage（可选持久化）
```

- **Mock 数据**：放在 `src/mock/[模块]/` 下
- **Service**：放在 `src/services/[模块].js`
- **数据持久化**：使用 `storage.js` 封装 + `storageKeys.js` 定义 key

## 7. 常见问题

### 7.1 下拉菜单被表格裁剪？

使用 `ActionBar` 组件，其「更多」菜单通过 `createPortal` 挂到 body，避免被 `overflow-auto` 容器裁剪。

### 7.2 如何新增模块？

1. 创建页面目录 `src/pages/[新模块]/`
2. 在 `DynamicSidebar.js` 添加 `navConfig` 配置
3. 在 `App.js` 添加路由 case
4. 如需 mock/service，在对应目录添加文件

### 7.3 如何保持筛选条件？

使用 `useLocalStorage` hook，将筛选状态持久化到 localStorage。
